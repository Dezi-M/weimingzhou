# 《未名舟》创作复盘：常见问题与改进

本文件只服务《未名舟》/ 命名权世界版项目，用于沉淀该项目创作过程中的内部反馈信号，不依赖用户逐条指出问题。

当同类听感、读感、追读问题在本项目的相邻章节、五章批次、十章复盘或重大修订中重复出现时，自动追加简短条目。不要把其他小说或项目的规则写进本文件。

本文件里的条目都只是软手感，不是硬校验。若某章反着写更有力，可以反着用；只要读者更想追、人物更像活人、场面更有身体感，就允许破例。

## 已沉淀手感

### 001 听感主线不能被资产和术语稀释

- 观察模式：单章能读懂，但连续听多章时，新资产、新规则、新职能连续出现，容易让听众忘记当前最想追的明面目标。
- 影响：读者会感觉故事一直在推进，却不确定“现在到底最该关心谁”。
- 下次调整：每章允许复杂，但要挂在一个可听主线下；先用旧物和口语目标稳住，再放正式名词。每个阶段持续回扣一个明面钩子，例如“救卿禾/追回童灯队/守住市井舱”。
- 保留/避免例：保留“锅、药箱、木牌、旧门”这类听得见的锚点；避免连续用正式名词当主要推进感。

### 002 提醒要像编辑坐在旁边，不像监工站在门口

- 观察模式：规则越补越完整时，容易把“提醒”写成“强制检查/高危报警/必须回收”，创作体验会变紧。
- 影响：系统虽然更安全，但作者会感觉被审视，章纲和审计也容易变成表格味，削弱直觉和手感。
- 下次调整：优先使用“航向观察哨、手感掂量、口吻提示条、旧账折痕、如果是我我会……”这类软输出；只说观察，不替作者下命令。
- 保留/避免例：保留“还记得这根线吗，不急，合适时轻轻碰一下”；避免“必须三章内回收”“高危遗忘项”“强制检查未通过”。

### 003 拒绝敌方诱惑后，要给读者一个更难但能走的活路

- 观察模式：合法港口、水袋、床位、药和安全身份这类诱惑很强，如果章节只写“我们不收”，容易变成道德姿态，而不是连续爽点。
- 影响：读者会认可主角立场，但身体上还在渴、饿、疼；如果没有新的具体动作，听感会悬空。
- 下次调整：拒绝之后顺手接一个正向动作：拖老街、找水缸、修门、分粮、开临时路、写一条小规则，或者让普通人拿旧物投一票。
- 保留/避免例：保留“陶老头守木牌后，邱成提出拖老街”；避免只停在“港口有毒，所以不能去”。

### 004 番茄写作技巧转译：每章都要让读者有下一滑的理由

- 观察模式：平台写作技巧反复强调第一页、章末钩、场景动作化、人物细节、铺垫-过程-余波。若《未名舟》只靠设定新或规则高级，读者仍可能在手机端划走。
- 影响：章节可能有内容，但缺少“我还想听下一段”的即时牵引；过渡章尤其容易变成搬运信息。
- 下次调整：每章先想“第一页抓什么、章末留什么”；场景跟着动作走；冲突双方都要有不能退的理由；高潮后至少留一处余波或物理残留。
- 保留/避免例：保留“半饼救人后露出第二只小手”；避免“事情解决了，大家理解了规则”。

### 005 判断是补阶段，还是重打地基

- 观察模式：有时卡住只是当前十来章缺少收束图；有时则是开篇类型承诺本身偏了。两者不能混成一种处理。
- 影响：若只是阶段卡顿就推翻主纲，会浪费已成立资产；若地基已偏还只补阶段图，会把旧问题越补越厚。
- 下次调整：先判断问题发生在哪一层。若是阶段缺钩，补轻量阶段图；若读者第一印象已从末世生存战滑向规则解题，则从 001 起重打承诺。
- 保留/避免例：保留“旧物、旧债、人物关系可以转成素材池”；避免把已被推翻的阶段补丁继续当方向盘。

### 006 读感不能靠章节外说明补线

- 观察模式：修入口和回声时，容易直接写“上一章发生过”“前一章那次”“后一章再回收”“本章要解决”“第八章那块登记牌”“第九十九章那次反杀”“第102章再回收”“这条线很重要”“读者会记住”等作者视角句，听起来顺，但读起来会出戏。
- 影响：读者眼睛会从场面里抬起来，看见作者在解释结构；前期沉浸感被打断，入口吸力下降。
- 下次调整：所有回声都尽量落到角色当下能摸到、听到、闻到的东西上，例如湿纸角、半个“到”字、空胃、糖水杯、旧校牌。需要解释时先给动作或物件，再给一句短判断。
- 语境限制：打斗、谈判、棋局、战术交换里的“下一回合/上一个回合/前两回合”属于场内词，不按破墙处理。
- 保留/避免例：保留“湿纸角贴在他胸口，像还带着老师沉下去前的体温”“锅底那块燃灯童登记牌还在他脑子里发烫”；避免“这句话在第二章刚刚发生过一次”“前一章那次提醒还在”“第八章那块登记牌还在他脑子里发烫”“第102章再回收”。

### 007 段落之间要像聊天一样接住

- 观察模式：为了补强入口、旧账和主题，容易在两个动作段之间插入一句漂亮判断，例如“这不是规则，是墙”“它比威胁更管用”。单句有力，但段落会像贴片，不像从上一段自然长出来。
- 影响：读感会出现轻微顿挫；读者能懂意思，却会感觉作者在场外帮忙总结，人物当下的呼吸被截断。
- 下次调整：段落转场优先用上一段留下的东西接下一段：一句没说完的话、一次咽口水、雨砸纸角、手腕一亮、锅盖一响、某人没接话。判断可以留，但要藏在身体反应或物件变化后面。
- 保留/避免例：保留“有人下意识舔了舔裂开的嘴唇，也有人看向白舟后面那排干净甲板”；避免孤零零写“它比威胁更管用”。

### 008 去AI味不等于堆辞藻

- 观察模式：修听感、读感、AI味时，容易把“更像人写的”误会成“更优美、更有修辞”，导致危险、饥饿、争吵和行动段被抹得太圆。
- 影响：文字看起来更精致，但场景温度会失真；该疼的地方不够疼，该急的地方不够急，普通人的话也会变得像旁白。
- 下次调整：按语境选语言。热场用短句、直话、错话、喘气；冷场和余波才给旧物、天气、气味和留白更多空间。审计时先问“这句顺不顺着上一口气往下走”，再问“它漂不漂亮”。
- 保留/避免例：保留“饿就说饿，疼就先骂一句”；避免把每个动作都改成带比喻的漂亮句子。

### 009 正文不要露出创作术语

- 观察模式：修爽点、钩子、章末和回声时，容易把工作台上的词带进正文，例如“伏笔”“铺垫”“爆点”“爽点”“镜头”“旁白”“转场”“主角/配角”。
- 影响：这些词会让读者意识到自己在看一个被设计的文本，沉浸感会比普通解释句掉得更快。
- 下次调整：正文只写读者能在场里看见或听见的东西。把“伏笔”改成湿纸角发热，把“爆点”改成白榜裂开，把“转场”改成黑水把船头推向另一片光。
- 语境限制：如果场内真的有投影、监控、广播屏幕，可以写“画面切到”；那是物件在动，不是作者在切镜头。
- 保留/避免例：保留“投影屏忽然亮了，画面切到江城市中心广场”；避免“镜头一转到江城市中心广场”“这里埋下一个伏笔”。

### 010 节点不要被新章节冲走

- 观察模式：写到 040、050 这种节点后，如果马上继续写 041、051，五章发布包、十章复盘或五十章卷审容易被“最新章”遮住，后续再回头会混批次。
- 影响：036-040 这种完整小循环可能没做标题/简介/听书节奏/Git 检查点；031-040 的连续性旧账也可能没复盘，导致下一批带着未结算的账继续滚。
- 下次调整：节点顺序固定为：未软审新章先补审；五章发布包/Git；十章连续性复盘；五十章宏观审计；再继续新章。若 041 已存在但 040 还没处理，先处理 036-040，041 留作下一批起点。
- 保留/避免例：保留“040 完成后先看 036-040 发布包，再审 031-040”；避免默认抓“最新五章”把 037-041 混成一个发布批次。

### 011 工程化记忆要轻，不要长成第二套大纲

- 观察模式：参考 Codex / GitHub Agent 写作工程时，容易照搬 `memory-bank`、硬门禁、评分卡和多阶段流程。项目会更像系统，但也可能和现有 `docs`、skill、阶段补纲重复。
- 影响：上下文变重，续写前准备过长；AI 会倾向于执行流程而不是听场景呼吸，段落也更容易变紧。
- 下次调整：只吸收“仓库级行为约定”和“当前故事状态账本”。`AGENTS.md` 管默认行为，`14_当前故事状态账本.md` 管最新事实；世界观、大纲、阶段补纲仍留在原文档里。写完新章只回写真正改变续写方向的事实，不把审计意见和写作课塞进状态账本。
- 保留/避免例：保留“042 后敌人拿到听源样本，043 自然转听源白税”；避免另建一套 `memory-bank` 再重复角色档案、世界规则和十卷纲。

### 012 三类留存风险要软配重

- 观察模式：连续高压解局章容易让卿宸显得永远正确，船上缺少低压生活，沈砚也可能因为太讲理而只像聪明对手、不像会伤人的反派。
- 影响：读者会认可设定和局面，但人物立体度、船上归属感、反派仇恨值会慢慢变薄；番茄追读不只看解局，还看读者想不想继续跟这群人待在一起。
- 下次调整：后续用软配重，不用硬插。卿宸可以因坚持正确原则而慢半拍，让卿禾、陆小满或普通人补上；连续 8-10 章外压后，顺手安排半章或一章饭、修船、认字、守夜、斗嘴；沈砚再亲自出场时，至少有一次“合法有效但残酷”的动作落到无辜者身上，而不只是温和论证。
- 保留/避免例：保留“卿宸不替人答导致救援慢半息，普通人自己敲钉补上”；保留“老周试做乡愁面失败，锅里却浮出下一章线索”；保留“沈砚把某个求救者合法修正成工具，语气仍平静”；避免把卿宸写成犹豫到全船重伤，那会从慢半拍变成降智；避免把低压写成纯水日常，或把沈砚忽然写成发疯式暴力反派。

### 013 严肃主线里要有破话和小算盘

- 观察模式：命名权、白钉、旧名、承约这些设定天然严肃，连续几章都写制度压迫和物理反杀时，读者会紧张但也会累，船上人容易像一直在战斗而不是一直在生活。
- 影响：故事有燃点，却少一口“我想继续待在这条船上”的轻松气；人物会从活人滑向功能队友。
- 下次调整：每 5-10 章轻轻放一个幽默风趣点，不写成段子。优先用破话、错话、小算盘、职业习惯和旧物尴尬：老周嘴硬，梁春枝损人，孙福来钥匙先露怯，邱成心疼碗，孩子认错字。笑点只给人喘气，不冲淡刀口。
- 保留/避免例：保留“邱成先问碗裂了谁赔，梁春枝骂他连哭都要称重”；保留“孙福来想藏票角，钥匙串先响，全船一起看他”；避免在刚夺名、刚封口、刚牺牲的第一口气里强行插笑话。

### 014 全员智商在线，困境来自取舍

- 观察模式：为了制造危机、解释设定或突出主角，容易让配角只会追问、敌人突然放水、主角错过明显方案，读者会觉得剧情在硬推。
- 影响：一旦读者觉得角色变笨，爽点会塌；再宏大的设定也会像作者在拨木偶，人物可信度下降。
- 下次调整：关键抉择前先看四件事：这个人知道什么、想保什么、怕失去什么、为什么不选更省事的路。错误可以保留，但要来自信息差、时间压力、身体代价、情感私心、旧伤或规则边界。敌人失败后，下一次最好学会一点反制。
- 保留/避免例：保留“沈砚算对承约漏洞，却没算到老周会为一口锅灰多骂一句，导致旧物回应提前”；避免“敌人明明能补钉却站着解释半章”“配角只问这是什么意思”“主角不做明摆着的方案却不给代价理由”。

### 015 节点检查要验真，不要重复阻塞

- 观察模式：最新章节越过五章/十章节点时，流程容易机械回头处理旧节点；但有些节点已经通过 Git 提交或发布包处理过，再回头会误判为 pending。
- 影响：能防漏账的流程会变成重复卡顿，作者明明已经完成 056-060 批次，却被 061 之后的“节点检查”拖回去重跑。
- 下次调整：遇到越节点章节，先看 `git log --oneline`、暂存区和是否已有发布包/复盘记录。若已有清楚批次提交，例如“完成056-060章代舵席小循环”，就把 061 视作下一批起点；若没有证据，再回头处理旧节点。
- 保留/避免例：保留“061 已存在，但 git log 已有 056-060 批次提交，所以继续审 061/设计 062”；避免只因最新章号是 061 就机械重跑 056-060。

### 016 单章闭环后再开下一章

- 观察模式：连续创作时容易一口气写出多章，虽然中间做了部分自查，但用户无法逐章确认“已审、已改、已入账”，也容易把五章节点挤到后面。
- 影响：章节质量控制会变得不透明；后续发现问题时要回滚多章语气和账本，复核成本更高。
- 下次调整：每章固定闭环：正文落稿 -> 单章审查 -> 必要小修 -> 命令检查 -> 账本回填/节点处理 -> 再决定是否开下一章。若最新章未审完，用户说“继续”也先补审，不直接写新章。
- 保留/避免例：保留“070 完稿后先审查并更新账本，再处理 066-070 节点，之后才进入 071”；避免“068、069、070 连续写完后才统一回头审查”。

### 017 短句密度要按场面调，不按数字硬压

- 观察模式：056-070 高压解局章短句比例普遍偏高，部分章节超过六成。短句能保住疼、急、纸字落下的节奏，但如果列表、过渡、物件描述也全被切碎，听起来会一直断拍。
- 影响：读者会觉得每一句都像鼓点，危急段有力，连续听多章会累；人物和环境少了自然呼吸。
- 下次调整：单章审查时顺手看短句密度和连续短句窗口。人物喊话、敌方纸字、名场面、章末钩可以短；物件描写、动作承接、非危急过渡尽量用中句接住。短句比例不是硬指标，高压章可以高，喘息章要主动降下来。
- 保留/避免例：保留“我想吃饭 / 我不想当灶”；保留“疼在 / 暖借 / 源不登”；避免把“锅盖、断秤、旧钥、绷带、缺口碗”每个都单独拆成一行，能合成一句时就让它们在一句里互相咬住。

### 018 规则战要降密，回到劫回地球的生存战斗

- 观察模式：旧稿中后段单章都有新旧物、新敌方记录和新反制，但连续读会形成固定鼓点：白纸落下 -> 旧物拆解 -> 白纸记录 -> 章末再抛更细规则。
- 影响：读者会承认脑洞聪明，却产生“又在解题”的疲劳；身体上不够渴、不够饿、不够怕船沉、不够想看下一场仗怎么打。卿宸容易像主持人，沈砚的脸被文书稀释，船外世界也会停在局部露一角。
- 下次调整：001 起重构时先定“哪块地球被拆走、怎么抢回来、慢一步失去什么”，再写命名权规则。连续规则反杀尽量不超过两章；每五章至少有一章让追船、守舵、分饭、修船、换药、残舟、押运船或黑航道地形站到台前。章末钩也要轮换，别总是“白纸记录”。
- 保留/避免例：保留“我想吃饭，不想当灶”这种人先于规则的钩；保留“锅盖补坏、残舟递钉、浮桥追船、沈砚落锤、船头偏航、黑水撞舷”这种可见动作；避免连续多章都靠“某规则待仿/待采/待核”吊下一章。

### 019 读者追的是破船抢回家

- 观察模式：命名权脑洞足够独特，但若正文先展示概念，读者会把它归类成高概念规则文；真正吸引追读的应是一群被写成材料的人在黑水末世里抢水、抢粮、抢药、抢名字，也抢回医院、学校、钟楼、菜市场和旧路，边逃边打，逐渐聚成船团和陆地。
- 影响：如果缺少资源压迫、持续危险、行动爽感、主角变强、队伍聚拢和活人日常，读者会佩服设定，却不一定惦记下一章。番茄读者尤其需要看见“再慢半步就完了”和“这一仗打赢后船真的多活了一口气”。
- 下次调整：每章章纲先写生存压力、押运/拍卖/追杀动作和抢回来的地球组件，再写命名权规则。每十章给一个可见大东西：残舟、战法、敌方武器、资源、临时岸、抢回来的街区组件。沈砚要有亲手落刀，卿宸要有抢笔压场，配角要从旧物持有人长成能打一段仗的人。
- 保留/避免例：保留“破船后面亮起第二艘、第三艘船”的长线追读感；保留“拉索、撞桥、登船、挡钳、抢人、补锅、抢药、抢水、夺舵、截押运船、拖急救舱”；避免把旧物反制写成答题、把敌人写成白纸自动出题、把配角写成只负责递道具。

### 020 允许从 001 起大修时，先重打读者承诺

- 观察模式：如果只从 036 或 041 往后改，后续能降规则战，但开篇读者仍可能先把本书归类为“高概念规则文”。现在既然 001 起允许重构，就不该继续沿用“前 35 章锁定”的补丁思路。
- 影响：开篇类型承诺会决定读者预期。若第一批读者没有先感到缺水、追兵、妹妹危机、黑水逃生和主角抢笔，他们后面看到再好的残舟、舰队、陆地，也可能已经带着“这是解题文”的滤镜。
- 下次调整：从 001 起把书型定成“命名权末世生存战斗爽文”。第一章先给公开羞辱和逃生动作，再给命名权定义；前十章先打出抢人、第一口饭、追踪、黑水逃亡和拒名船身份；每十章结算一个可见资产。
- 保留/避免例：保留“赵成被写成示材台”“卿禾燃灯童”“第一顿非法粥”“黑航道第一战”这些强入口资产；避免让第一卷主要靠“理解命名规则”推进，避免再把旧锁定区当不可触碰的地基。

### 021 旧物不是纪念品，抢回地球组件要立刻变成战力

- 观察模式：把本书调成末世生存战斗后，仍可能写成普通追船、抢水、打怪；命名权脑洞又可能退回概念展示。真正更锋利的口径应是：地球正在被万界拆成货，未名舟每抢回一块地球，都要让它重新长出人的用途。
- 影响：如果“锅、学校门、钟楼、急救室、菜市场青石”只负责怀旧和煽情，读者会觉得旧物有温度但缺爽感；如果抢回来的东西不能改变饭、药、路、火、舱、索、阵地或战法，宏大会停在口号上。
- 下次调整：章纲先问四句：哪块地球正在被拆、卖、押运、登记或改用途？谁会因此立刻饿、疼、死、失去亲人或选择？未名舟用什么动作抢回来？抢回来后立刻变成什么生活能力或战斗能力？命名权只从这个过程里露，不单独开课。
- 保留/避免例：保留“白灯舰队围剿第一顿非法饭”“急救方舱被拖下拍卖链后立刻救伤员”“钟楼声被抢回后让黑航道短暂现路”“菜市场青石铺成第一块临时岸”；避免“抢回旧物后大家感慨一番”“解释某物象征故乡但不参与战斗”“敌方只出规则题而不押运、验货、拍卖、追杀”。

### 022 标题要跟着重构重新掂

- 观察模式：章节正文重构后，旧标题可能还停留在旧引擎上，例如“船内公议”“合法港口”偏概念，而新版正文真正的目录钩已经变成“两瓶水分不下”“身有钉，话未定”。
- 影响：标题是番茄目录里的第一层拉力。正文已经有物件、伤口和动作，标题却还抽象，会少一口点击欲，也容易把读者误导成规则/会议章节。
- 下次调整：每章重构后顺手复看标题。优先选本章最能被读者一眼看见、能复述的物件或动作；少用抽象概念词，除非这个概念本身已经成了高辨识度名场面。
- 保留/避免例：保留“她不叫燃灯童”“七十张嘴”“第一块求救木牌”“身有钉，话未定”；避免正文已变成战斗/救援后仍沿用“公议”“雏形”“合法港口”这类说明型标题。

### 023 五十章后先加地基钉，再追求大重写

- 观察模式：001-050 已经形成正史和商业循环，问题不是整体失败，而是少数地基承诺需要更早、更硬地钉住：万界验货、沈砚亲手合法伤人、卿宸压场、主动抢资源、残舟有脸。
- 影响：如果直接大拆，会损伤已成立的饭线、医舱、兵骨、老街和残舟；如果只继续往后写，又可能把机制疲劳带进 051-060。
- 下次调整：先做高杠杆小改：开篇多一处货单/验货，反派多一处亲手合法伤害，节点章多一处主角调度或可见大资产。然后按十章小循环逐批审，不把五十章一次打碎。
- 保留/避免例：保留“001 同类急救舱进入首轮验货”“004 临时浮楔”“020 战场中枢式掌舵”；避免为了显得大修，把已经好用的第一顿粥、苏晚、铁牙、老街、残舟整段推翻。
