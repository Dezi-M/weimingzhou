# 《未名舟》正文写作与审计手册

本文件记录当前正文写作与审计口径，只服务正在推进的新大纲。

## 0. 使用口径

本表不是逐条扣分表，而是创作诊断工具。

执行时分三层：

1. 红线：读者看不懂、人物不像人、章节没有推进、正文格式不适合发布。这类问题优先修。
2. 诊断：人味、意外钩、感官、私密锚点、爽点轮换。发现缺口后先判断是否真的影响追读。
3. 建议：具体频率、数量、模板、复盘项。可按章节类型灵活取舍。

如果某条规则让正文变僵，优先相信场面和人物，再回头调整规则。

去AI味、听感、读感都不是“辞藻更优美”。危险、饥饿、争吵、疼痛和行动段可以直白，甚至应该直白；旧物、余波、沉默和情绪回声才更适合多一点质感。

## 1. 正文格式

正文只写纯 TXT，不混入 Markdown。

不得使用：

- `#` 标题符号。
- `**` 加粗符号。
- Markdown 表格。
- 作者备注、HTML 注释或审计说明。

字数参考：

- 普通推进章：常见 3000-3800 字。
- 关键转折章：常见 3800-5200 字。
- 小高潮章：常见 5200-7000 字。
- 大高潮或卷尾：常见 7000-9500 字。

以上只作为听书节奏与发布节奏参考，不作为硬性通过条件。场面压力、情绪回收、动作清晰和章末钩优先。

## 2. 开写前八格

真正写正文时，先只看这一页。

1. 危险：本章谁快失去什么。
2. 目标：本章要抢、救、守、骗、换、逃的东西是什么。
3. 敌压：敌人主动做了什么，不是只等主角行动。
4. 行动：主角方用什么具体动作推进，不靠解释。
5. 战利品：本章得到或失去什么可见物。
6. 感官：命名、抢笔或船体变化用哪一种感官呈现。
7. 关系：本章至少一组人物关系有什么微小变化。
8. 钩子：章末是危险钩、奖励钩还是反转钩。

如果八格空得厉害，先补纲，别急着写正文。

## 2A. 软规则创作线

开写前不需要把所有规则背一遍，只抓一条软线：

脑洞火花 -> 单章八格 -> 正文动作链 -> 章末物理钩 -> 审后自修 -> 必要时回写账本或复盘。

每一环只问最有用的事：

- 脑洞火花：这一章有没有一个比旧方案更刺激、更有辨识度的可能。
- 单章八格：危险、目标、敌压、行动、战利品、感官、关系、钩子是否能支撑正文。
- 正文动作链：人物是否一直在做事，而不是旁白解释。
- 章末物理钩：本章落下的是物、伤、敌招、船体变化、救回的人，还是抽象总结。
- 审后自修：发现影响追读的硬问题，先改正文，不把问题留给下一章。
- 回写进化：如果这是可复用问题，写进账本、伏笔索引或创作复盘。

软规则的意思不是“可做可不做”，而是它服务场面。场面热时，规则退后；场面散时，规则把路找回来。

## 3. 规则战降档，生存战斗升档

这是当前最重要的书型校准。

命名权脑洞要保留锋利度，但章节快感不能主要变成“纸面机关出题 -> 旧物拆题 -> 纸面机关记录新题”。

每章开写和审稿时，优先问：

- 这章他们缺什么：水、粮、药、床、热、船板、航路、时间，还是一个快被押运走的人？
- 这章他们抢什么：人、药柜、水柜、旧门、钟声、街口、残舟钩、敌方武器、地球组件，还是一条能活下去的路？
- 慢一步会失去什么：伤者断气、卿禾灯芯变薄、船头偏向白灯、旧物被归档、残舟被拖走、临时岸沉下去？
- 抢回来以后立刻改变了什么：能煮饭、能压伤、能挡风、能守舷、能开路、能拉索、能并肩、能让船多活一口气？

一句话判断：

> 读者这一章记住的最好不是“我懂了一个规则”，而是“他们差点渴死，抢回三滴水”。

## 3A. 039 后新引擎：人间开阵写法

“人间，开阵”是阶段爆点，不是每章口号。

正文原则：

1. 不先解释终相、复相、万象开阵。先写敌人压境、旧物受损、人做选择、战场改变。
2. 人是主导，旧物是证据。必须先有人做反用途动作，旧物才应阵。
3. 敌人要聪明，会拆人和旧物之间的关系，而不是站着等旧物反杀。
4. 开阵要有失日债。锅、床、校牌、车站、门、碗、秤被迫上战场后，必须留下回不去日常的伤。
5. 小战不喊完整口号，中战可半句，大结算才完整喊。

每次开阵前，先确认四件事：

- 谁被压成了唯一用途。
- 谁先做了一个不服从用途的动作。
- 哪件旧物、旧业、旧城或旧伤应了这个人。
- 赢了什么，欠了什么。

战斗结构建议：

1. 压境：敌人拿出聪明方案，让读者觉得难打。
2. 小胜：七奇锋或某个普通人撕开一个口子。
3. 反噬：敌人预判，小胜变陷阱。
4. 人心动作：一个人拿出最舍不得的东西或做最疼的选择。
5. 开阵破局：只赢关键一段，不全赢，留下更大敌人和失日债。

判断一句：

> 爽点必须带泪，泪点必须反推爽。

如果只是旧物显灵、卿宸解释、敌人被无脑碾压，本章必须重搭。

## 3B. 上帝视角防滑坡

039 后每章审稿时，额外看五个滑坡点：

1. 是否把“人间开阵”写成新术语，而不是写成抢人、挡脚、扣影、救伤、断门、开路。
2. 是否把阶段奖励拖太远。本章哪怕输了，也要给读者一个新样本、新战法、新伤口、新关系或新敌方误判。
3. 卿宸是否有主角动作。他可以克制，但不能旁观；至少要挡、看、错开、递回、压住或亲自承受一次。
4. 敌人是否在学习。上一章吃过的亏，下一次要被它利用、污染、伪装或反设陷阱。
5. 奇锋和军团是否混用。奇锋是小队破点，军团是大面战线；不要把小队写成低级军团，也不要让军团做只有三个人潜入才能做的活。

如果五项里有两项空，本章不是润色问题，而是发动机问题，优先重搭场景。

## 4. 人味、流畅与去AI化

角色不能像规则容器。

主要出场人物尽量至少有一种活人表现：

- 身体反应：饿、冷、疼、困、手抖、嘴硬、忍笑、喘不上气。
- 小动作：藏东西、摸旧物、挡一下、退半步、嘴上骂手上帮。
- 不漂亮的话：急了说错话、怕死、嘴碎、护短、算小账。
- 私人判断：不是因为大道理，而是因为他在乎某个人、某口饭、某件旧物。

正文要让手机读者一滑就懂，听书读者一听就跟上：

- 大约每300-500字落一次具体动作、物件、声音、气味或身体感。
- 避免连续三段全是解释、判断、总结。
- 设定尽量从动作里出来：先看见人被改、饭被锁、船被拖，再说规则。
- 对话要推进局面，不能只是互相解释世界观。
- 每章尽量有一个“读者能复述的朴素句子”。

## 5. AI腔软雷达

以下情况出现两项以上，优先回看；如果已经影响顺读和人物真实感，再重写：

- 连续排比，句式过齐。
- 用抽象词替代画面，如“秩序崩塌”“情绪爆发”“规则升级”，但没有人、物、动作。
- 角色说话像总结报告。
- 段尾总是拔高主题。
- 主角频繁自我审判，却没有立刻变成行动。
- 旁白替读者解释“这意味着什么”，而不是让场面自己说明。
- 每个人说话都同一种口吻。
- 章末强行升华，没有物件、动作、声音或选择后果。

修稿顺序：

1. 把抽象判断换成具体动作。
2. 把旁白解释换成角色选择。
3. 把漂亮台词换成带脾气、带职业、带饥饿或疼痛的话。
4. 把总结段拆进饭、灯、船、伤口、旧物、敌人动作里。
5. 把“因为X所以Y”改成“X触到实体 -> 实体变化 -> 人被迫反应 -> 局面改变”。

## 6. 角色口吻

角色台词要长在职业、旧物和脾气里。

- 老周尽量从锅、饭、碗说起。
- 苏晚从伤口、位置、药和记录说起。
- 铁牙从守线、打结、旧役说起。
- 陆小满从木牌、铅笔、记账说起。
- 云织从针脚、门、线说起。
- 孙福来从钥匙、门缝、后路和旧城说起。
- 梁春枝从秤、账、斤两和不公平说起。

配角不要被写成提问机器。可以问错、怕错、站错一息，但要有自己的职业判断和利益算盘。

## 7. 情绪呼吸与笑点

《未名舟》不能只有高频爽点，也要有让读者想住在船上的日常温度。

情绪缓冲不等于水章。它是下一次爆发的情绪蓄电。

可用呼吸方式：

- 半页分水、补锅、换药、缝衣、修门。
- 角色嘴硬、错话、小算盘、职业习惯。
- 旧物尴尬：锅盖漏、钥匙卡、断秤偏、碗不够大。
- 饭香、药味、旧校牌、错字本、乡愁面。

幽默口径：

- 不要写段子。
- 不要冲淡死亡、背叛、名丢失或新牺牲。
- 笑点来自人还活着：老周嘴硬，梁春枝损人，孙福来钥匙先露怯，邱成心疼碗，孩子认错字，旧工具在最糟糕的时候掉链子。

## 8. 命名感官

每章涉及命名、抢笔、写骨、定名、拆名时，优先呈现一种具体感官，但按场面规模调节强度。

可选：

- 视觉：淡金铭文膜、锈迹、灯芯纹、黑封条。
- 听觉：钢笔刮纸、印章落下、旧名低语、船骨低鸣。
- 触觉：烧红铁丝般刺痛、断钉烫掌、木纹爬肤。
- 嗅觉：焦味、火油味、旧纸味、药箱木屑味。

避免只写“他看见裂缝”“他抢回一笔”。

感官不要为了漂亮而堆满。若这段本来靠对话、动作或旧物成立，轻轻一烫、一声纸响、一点焦味就够。

## 9. 每章爽感软检

每章优先命中两项；若是低压日常、情绪余波或必要过渡章，可以用强情绪、强关系变化或强章末钩替代。

- 抢名。
- 救人。
- 打脸。
- 反杀。
- 收伙伴。
- 开船舱。
- 得旧物。
- 接承约。
- 分账。
- 军团组合。
- 敌人阶段胜利。
- 普通人主动选择。
- 船体升级。
- 地图奇观。
- 人间开阵。
- 敌人聪明反制后被艰难打穿。

若只命中“解释规则”，本章容易空，优先改成动作、选择或可见战果。

每章尽量明确一个“本章战利品”或“本章损失物”。它可以很小，比如一笔真名、一枚断钉、一张旧票、一碗粥、一条航迹、一个人转身留下。

## 10. 章末三钩

章末优先命中一钩：

1. 危险钩：敌人来了、倒计时变短、某人被标价。
2. 奖励钩：新船舱亮了、新伙伴醒了、新物件到手。
3. 反转钩：刚赢的东西被敌人拿去反用，或被救者提出意外选择。

最好一章末尾同时有奖励和危险。只留抽象命题时，优先补一个物件、动作、声音或敌方反应。

## 11. 每章读者审计

写完每章后快速扫：

1. 普通读者能否一句话说出本章发生了什么。
2. 本章有没有人命、饭、灯、船、旧物、敌人脸、任务中的至少一项。
3. 本章是否正义一直爽，还是变成主角原地自责。
4. 本章敌人有没有聪明反制。
5. 本章是否获得或损失了一个可见资产。
6. 本章章末钩子是否具体。
7. 本章有没有术语先行。
8. 本章有没有 AI 腔。
9. 本章是否和前后文连续。
10. 本章意外钩是什么：表面用处、真正用处、代价、回收章分别是什么。
11. 本章新规则是否先见血、先损物、先改变局面，再给解释。
12. 本章章末是否有物理结算：多了什么、少了什么、破了什么、敌人学会了什么。
13. 本章主角、配角、敌人的选择是否智商在线。
14. 若涉及开阵，是否是人先动、物再应，而不是旧物自动开挂。
15. 本章胜利是否留下失日债，而不是白拿爽点。
16. “人间，开阵”是否只用在足够大的阶段爆点上。

以上问题用于发现风险，不是逐条打勾。

## 12. 完整审计输出模板

正式写章和重修关键章可参考八块：

1. 八格驾驶舱：危险、目标、敌压、行动、战利品、感官、关系、钩子。
2. 意外钩审计：获得什么、谁带来、表面用处、真正用处、代价、回收章、情绪推进。
3. 基础发布检查：字数、纯 TXT、无 Markdown 误入、短句是否有效、番茄复制是否方便。
4. 读者层检查：快读读者、听书读者、情绪读者、设定党、老书虫、反感 AI 读者是否能读进去。
5. 核心检查：开章五秒、三分钟目标、八分钟奖励、章末三钩、爽点或情绪落点、可见资产或损失。
6. 创作质量检查：活人感、去 AI 化、阅读流畅度、生活细节、非漂亮话、人物是否像活人。
7. 连续性检查：是否承接上一章，是否为下一章留下债务，人物能力是否错配。
8. 主创主动升级：本章新增了什么创意，哪里可能俗，哪里还能更爽，是否需要立刻改正文。

若审计发现影响追读、人物真实感或连续性的实质瑕疵，优先先改正文，再进入下一章。

## 12A. 审后自修与自主决策权

审计不是写评语，是给正文动刀。

默认权限：

1. 句子级问题：直接改。包括 AI 腔、解释过长、口吻不稳、动作不清。
2. 场景级问题：可以重搭。包括开头没压力、敌人不主动、反杀太容易、章末太虚。
3. 章节级问题：可以重写发动机。包括本章主菜变成规则说明、卿宸旁观、普通人没选择、胜利无代价。
4. 阶段级问题：先暂停继续写，回到 `03` 重排五章或十章路线。
5. 地基级问题：回到 `00/04/05` 改主控、能力或审计规则，不把坏规则继续执行。

什么时候不自行改：

- 用户明确说“只审不改”。
- 两条路线都能成立，且会明显改变主线口味。
- 需要决定是否推翻已发正文。
- 涉及用户明确保留的角色、设定、章节事实。

审后最小闭环：

1. 发现问题。
2. 判断层级。
3. 改正文或改章纲。
4. 再扫一次入口、动作链、章末钩。
5. 只把会影响未来续写的事实写回账本。
6. 若问题会反复，写进创作复盘，让下一次自动避开。

一句话：

> 能修句子就修句子，句子救不了就重搭场景，场景救不了就重写本章，章节救不了就改路线。

## 13. 每五章/十章/五十章

每五章看重复：

- 最近五章有没有同一种敌压、同一种解法、同一种文书结尾。
- 若重复，下一章主动换菜：资源、追击、残舟、日常喘息、主角压场、普通人高光。

每十章通读：

- 剧情是否连续。
- 角色能力是否错配。
- 爽点是否轮换。
- 有没有资产结算。
- 有没有新钩子。
- 有没有读者能记住的名场面。
- 有没有敌方经济诱惑。
- 有没有一个角色主动欲望变强。

每五十章卷审：

- 商业循环是否完整。
- 反派是否升维。
- 七类资产是否结算。
- 普通人是否改变战局。
- 赢大局输小局是否成立。
- 下一循环主菜是否清楚。

## 14. 百万读者模拟

这是模拟读者团，不是真实读者。

每个大阶段按以下读者层过滤：

- 快读爽文读者：看是否够刺激、够清楚、够想翻页。
- 听书读者：看是否听得懂人物、目标和战果。
- 设定党：看体系是否新。
- 情绪读者：看妹妹、伙伴、普通人是否有情绪抓手。
- 商业读者：看章末是否有追读欲。
- 老书虫：看是否套路过熟。
- 反感 AI 读者：看是否有机械味、概念味、空泛味。

若七类读者中有三类以上排异，优先先改影响最大的入口、目标或章末钩。

## 15. 主创主动审计

每章写完后，Codex 额外看一眼：

1. 本章有没有主动新增的创意，而不是只执行已有想法。
2. 本章有没有一个能让读者眼睛亮一下的陌生感。
3. 本章有没有一个可长期复用的基建资产。
4. 本章有没有一个可传播的名场面、物件或台词。
5. 本章有没有一个敌人更聪明的新打法。
6. 本章有没有一个角色更活的新动作。
7. 如果我是严苛主编，我会要求本章再升级哪里。

若以上七问有三项以上答不出来，优先补一个能立刻改善追读的创意、基建或爽点，再进入下一章。
