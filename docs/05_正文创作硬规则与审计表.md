# 《未名舟》正文创作软规则与审计表

## 0. 使用口径：规则服务故事

本表不是逐条扣分表，而是创作诊断工具。执行时分三层：

1. 红线：读者看不懂、人物不像人、章节没有推进、正文格式不适合发布。这类问题优先修。
2. 诊断：人味儿、意外钩、感官、私密锚点、爽点轮换。发现缺口后先判断是否真的影响追读。
3. 建议：具体频率、数量、模板、复盘项。可按章节类型灵活取舍。

如果某条规则让正文变僵，优先相信场面和人物，再回头调整规则。

去AI味、听感、读感都不是“辞藻更优美”的同义词。危险、饥饿、争吵、疼痛和行动段可以直白，甚至应该直白；旧物、余波、沉默和情绪回声才更适合多一点质感。审稿时先看一句话是否顺着上一口气往下走，再看它漂不漂亮。

## 1. 字数参考

普通推进章：常见 3000-3800字。
关键转折章：常见 3800-5200字。
小高潮章：常见 5200-7000字。
大高潮或卷尾：常见 7000-9500字。

以上只作为听书节奏与发布节奏参考，不作为硬性通过条件。场面压力、情绪回收、动作清晰和章末钩优先。

宁可少写解释，也优先别少写动作、选择、战果和章末钩子。

## 2. 手机端排版

短句可以用，但要有力量。

允许单独成行：

- 重大反转。
- 角色当场崩裂或爆发。
- 战斗动作重击。
- 章末钩子。
- 一句足以让读者停住的台词。

少让无意义短句单独成行：

- 普通编号。
- 普通说明。
- 半句解释。
- 为了制造AI式节奏而拆开的句子。

例：`D-019。` 这种编号不需要单独一行，除非它是当章核心反转。

## 3. 人味儿、流畅与去AI化原则

这些不是给正文套枷锁，而是给修改时用的诊断工具。故事、人物和当章情绪优先；规则只负责提醒哪里可能变硬、变空、变像AI。

不要为了“人味儿”故意把句子弄糙，也不要为了“高级感”把每段都修成漂亮展示。语境能承受什么，就用什么。

正文优先做到：

每章尽量有生活细节。  
尽量让至少一个人说一句不漂亮的话。  
尽量让至少一个动作不是为了讲设定，而是为了活着。  
尽量避免连续概念句。  
尽量留下一个能被听书读者记住的具体物件。

### 3.1 人味儿原则

角色不能像规则容器。主要出场人物尽量至少有一种活人表现：

- 身体反应：饿、冷、疼、困、手抖、嘴硬、忍笑、喘不上气。
- 小动作：藏东西、摸旧物、挡一下、退半步、嘴上骂手上帮。
- 不漂亮的话：急了说错话、怕死、嘴碎、护短、算小账。
- 私人判断：不是因为大道理，而是因为他在乎某个人、某口饭、某件旧物。

情绪不要直接贴标签。少写“他很愤怒”“她很感动”，优先写他做了什么、没做什么、把什么东西攥紧了。

### 3.2 流畅原则

正文要让手机读者一滑就懂，听书读者一听就跟上。

- 大约每300-500字落一次具体动作、物件、声音、气味或身体感。
- 尽量避免连续三段全是解释、判断、总结。
- 设定尽量从动作里出来：先看见人被改、饭被锁、船被拖，再说规则。
- 对话要推进局面，不能只是互相解释世界观。
- 句子长短跟场面走：动作和危险用短句，情绪和日常允许长一点，但不能绕。
- 每章尽量有一个“读者能复述的朴素句子”，如“这不是饭，这是狗链子”“药归箱，人归人”。

### 3.3 AI腔软雷达

以下情况出现两项以上，优先回看该段；如果已经影响顺读和人物真实感，再重写：

- 连续排比，句式过齐。
- 用抽象词替代画面，如“秩序崩塌”“情绪爆发”“规则升级”，但没有人、物、动作。
- 角色说话像总结报告。
- 段尾总是拔高主题。
- 主角频繁自我审判，却没有立刻变成行动。
- 旁白替读者解释“这意味着什么”，而不是让场面自己说明。
- 每个人说话都同一种口吻，没有年龄、职业、脾气差异。

### 3.3.1 AI味软雷达

这一组只做修稿雷达，不做硬性扣分。角色台词、命名庭公文、合法港口播报、伤情牌记录等场景可以故意保留一点“说明书味”，但正文旁白和主角团对话要尽量避开机械感。

优先纳入以下软规则：

- 少用路标套话。正文尽量不用“首先、其次、最后、综上所述、值得注意的是、毋庸置疑”。需要列步骤时，用动作自然带出，或让角色边做边说。
- 少用万能开头和空泛评价。避免“在当今时代”“意义深远”“引人深思”“不可或缺”等空话。要评价，就落到人、物、代价和下一步动作。
- 少用工整书面句。警惕“不仅……而且……”“然而”“既要又要也要”成串出现。需要转折时，优先用角色反应、物件变化或一句不漂亮的话。
- 少写完美三段论。章节和段落优先按“场景 -> 冲突 -> 选择 -> 后果”推进，不为了整齐而总分总。
- 少做强行辩证。可以有反方和代价，但要写清楚谁承担、现在怎么选，不要只写“同时也要兼顾”。
- 少解释常识。读者能从画面懂的，不再补定义。比如先写港口水袋、受领小字、孩子咳嗽，再让读者感到“合法港口诱惑”。
- 少用客服腔。避免“建议您、请确保、温馨提示”式口吻，除非它来自敌方文书或系统播报。
- 少用安全形容词。深刻、重要、复杂、显著、全面等词出现时，后面要有证据锚点；没有锚点就删。
- 少堆金句和四字词。金句不替代情节，四字词连续出现时至少改一个成大白话。
- 少视角漂移。旁白、角色感受、公文播报要分清。不要一会儿上帝视角，一会儿“我们可以看到”。
- 保留一点人类瑕疵。允许短句、停顿、嘴硬、算小账、重复一句脏话或笨话。只要符合人物，就比完美句子更好。
- 结尾不强行升华。章末优先留物件、动作、声音或选择后果，不用“这具有重大意义”。

修稿时只抓最影响阅读的1-3处，不要为了反AI味把所有句子改得故意口水。小说可以有利落、干净、漂亮的句子，关键是它们要长在人物和场面里。

### 3.3.2 长篇小说专用软雷达

这一组专门用于《未名舟》正文，不做硬性逐条检查。每章写完或重修时，只挑最刺眼的1-3处处理；如果某一条为了角色、节奏或敌方文本故意保留，可以保留。

- 人物别过度讲理。真实人吵架会绕、会急、会嘴硬、会说错。若每个人都像辩手，优先打断逻辑，补一个动作、偏见、误会或算小账。
- 角色口吻别平均。老周尽量从锅、饭、碗说起；苏晚从伤口、位置、药和记录说起；铁牙从守线、打结、旧役说起；陆小满从木牌、铅笔、记账说起；云织从针脚、门、线说起。
- 道具别只当按钮。锅、校牌、门、木牌、牙套、腕带、绷带、钟声都要有磨损、脏污、重量、旧主人或被人藏过的痕迹。
- 情绪别太准。人物未必立刻知道自己为什么痛。优先写手停了、碗没接稳、话说半截、眼神避开，再决定要不要点明。
- 牺牲别太漂亮。主动牺牲的人也可以怕、骂、不甘、算账、后悔一秒。太顺的牺牲容易像模板。
- 反派少讲世界观。沈砚和命名庭优先用水、药、港口、保命书、通缉页来证明自己，不要每次靠长篇解释压人。
- 章末钩别总工整。少用“就在这时，异变突生”一类口吻。优先用物件变化收尾，如钟慢半拍、碗口朝里、木牌多一行、绷带渗白痕。
- 比喻别太密。关键处可以漂亮，连续漂亮就假。动作章多用硬动作、短句和白话，情绪章再允许一点细比喻。
- 正义话别说太满。卿宸可以沉默、说错、被普通人问住、被卿禾或陆小满补上。主角越总是正确，越像AI替他说答案。
- 全员智商要在线。主角、配角、普通人、敌人都可以犯错，但错误要来自信息不足、时间压力、身体代价、情感私心、旧伤或不得不做的取舍，别来自“突然想不到明摆着的方案”。如果角色不选最简单的路，正文最好让读者看见那条路为什么走不通、太贵、太脏，或已经被敌人预判。
- 敌人失败别靠降智。沈砚和命名庭可以输，但最好输在信息差、旧物反噬、普通人的意外选择、成本算错、规则边界被抢，而不是明明能赢却废话、放水、忘记补刀。聪明敌人输一次，下一次要学会一点东西。
- 配角不要被写成提问机器。陆小满、苏晚、老周、铁牙、云织、孙福来这些人可以问错、怕错、站错一息，但要有自己的职业判断和利益算盘。别让他们只负责问“这是什么意思”来帮作者解释世界观。
- 失败要有后遗症。小胜之后至少留一处痕：记忆少一块、绷带少半卷、木牌多一行、某人不再全信主角、敌方学会反制。
- 信息少从旁白倒出。优先让箱盖小字、伤口、锅底、门缝、钟声、保命书、木牌把信息带出来。
- 普通人别工具化。船民不能只喊“支持”或“反对”。尽量给他一只碗、一个亲戚、一次怕死、一次说对、一次说错。
- 命名权世界的规则要有手感。规则不要只在台词里成立，最好能碰到皮肤、勒进腕骨、压弯船舷、改变一碗水的归属。

### 3.3.3 讲道理与说明化软雷达

这一组专门防止正文变成“作者讲规则”“角色讲道理”“旁白说明会”。它只做修稿提醒，不做硬性禁令。敌方公文、命名庭判词、木牌记录、伤情牌、承约条目可以有说明感，但正文叙事和主角团对话要尽量让规则从场面里长出来。

- 规则先见血，定义往后挪。不要先解释“重力加倍”，先写骨头断裂声、碗被压裂、手腕被勒出白痕。解释优先在受伤、失物或局面改变之后出现，且尽量短。
- 道理尽量寄生在损耗品上。要表达“不能代答”，优先写铅笔压下又收住、门没有被替别人关上、锅补了三次又漏，而不是旁白反复说“这是代答”。旧物最好有缺口、锈迹、修补处或被人藏过的痕迹。
- 拒绝说明书式新设定。承约、舰队、未名岸、听源、终名这些大概念，每章只露当章用得上的一角，先让一个人被它害了、救了或卡住，再让角色说这叫什么。
- 情绪先于逻辑。读者先看见人疼、怕、饿、误会、后悔，再接受规则名。不要先下定义，再补一个受害例子。
- 解释尽量短。连续说明写长时，优先插入动作、物件、声音、气味、外部变化或角色打断。没有动作承托的连续说明，优先删。
- 因果链尽量物理化。把“因为X所以Y”改成“X触到实体 -> 实体形变或位移 -> 某人受迫反应 -> Y确立”。例如白钉碰碗沿，碗底裂出白痕，邱成护碗后退，试岸归属被迫挂账。
- 逻辑最好改变物理环境。讲理段若没有让白线偏、木牌亮、敌人改招、灰被震落、某人敢动或某物破损，就优先删掉，换成敌人提前动手。
- 角色是赌徒，不是辩手。台词要服务欲望和筹码：求生、护人、贪财、复仇、保饭、保旧物。哪怕讲理，也要被职业动作打断：老周翻锅，苏晚压伤，铁牙绕索，陆小满记账，云织拨线。
- 主角要背负错误成本。卿宸可以犹豫、被问住、冒出省事的坏念头，但不能站着想半章。同一场景内优先用动作回答：救人、砸钉、开火、退一步、交出选择权，或承担选错后的伤。
- 主角犹豫不能冷场。被问住时，手里还要做一件具体事：拆绳、擦血、翻木牌、压舵、数药瓶。想的同时在干，读者才会觉得他在扛，不是在卡。
- 反派是投毒者。少喊理念，多递礼物：掺沙的米袋、缺员的名单、倒计时的灯、带小字的水袋、巡听灯。让道具替反派发言，反派只需微笑或补一刀。
- 作者别跳出来当导游。“这意味着”“本质上”“由此可见”“说白了”这类句子容易刺眼，除非是角色急口骂人，否则优先删。相信读者能从事实里补全。
- 断章即结算。章末尽量留下物理状态改变：手里多了东西，身上添了伤，木牌多一行，锅盖裂一块，敌人学会一招。不要只落在心理感悟或会议结束。
- 拒绝无痛爽感。爽点不能只写成结论，要伴随实体的破坏或建立：白钉退、锅盖裂、钥匙卡住、门槛发热、敌方判词失效。
- 情绪尽量带杂质。单一愤怒、单一喜悦容易假。愤怒里可以有恐惧，喜悦里可以有后怕，胜利后可以先心疼碗、药、锅盖或绷带。
- 环境尽量是参与者。黑水、冷光、锅烟、湿布帘、药味、热气不要只当背景。它们可以影响判断、动作或身体，比如汗水刺眼让刀偏半寸，冷水让绳结变硬。
- 信息差尽量制造悬念。读者可以比角色多知道一点危险，但角色不能长期比读者知道更多。若保留秘密，优先用物件异常、反常动作或敌人反应先给读者抓手。
- 名场面要可传颂。每1-3章尽量留下一个不用解释也能复述的台词、动作或物件定格，如“这火不烧人，烧名”、锅盖借影、半块饼压住求救布。
- 讲理段保留标准：它最好改变局面。说完以后，白线偏了、木牌多字了、某人敢动了、敌人改招了，这段才值得留。
- 一句判断：这段如果抽掉解释还成立，解释可以留一点；如果抽掉解释读者就看不懂，说明解释没绑在动作和物件上，先改动作，再写解释。

一句总心法：小说去AI味，不是把文字故意写粗，而是让每句话都像从某个人的身体、职业、旧物和当下处境里长出来。

### 3.4 修改优先级

发现AI腔时，按这个顺序改：

1. 把抽象判断换成具体动作。
2. 把旁白解释换成角色选择。
3. 把漂亮台词换成带脾气、带职业、带饥饿或疼痛的话。
4. 把总结段拆进饭、灯、船、伤口、旧物、敌人动作里。
5. 把“因为X所以Y”改成“X触到实体 -> 实体变化 -> 人被迫反应 -> 局面改变”。

## 4. 听书黏性

1分钟钩：开头最好有危险或异常。  
3分钟粘：尽快给出本章目标。  
8分钟奖：中段尽量有小胜、小反转或新资产。  
章末挂：结尾要尽量让读者想听下一章。

听书时人名、能力、敌人和目标要清楚，不能靠表格理解。

## 5. 每章爽感软检

每章优先命中两项；若是低压日常、情绪余波或必要过渡章，可以用强情绪、强关系变化或强章末钩替代：

- 抢名。
- 救人。
- 打脸。
- 反杀。
- 收伙伴。
- 开船舱。
- 得旧物。
- 接承约。
- 分账。
- 军团组合。
- 敌人阶段胜利。
- 普通人主动选择。
- 船体升级。
- 地图奇观。

若只命中“解释规则”，本章容易空，优先改成动作、选择或可见战果。

额外落点提醒：每章尽量明确一个“本章战利品”或“本章损失物”。它可以很小，比如一笔真名、一枚断钉、一张旧票、一碗粥、一条航迹、一个人转身留下。没有战利品或损失物时，要确认这一章是否仍有清楚落点。

额外追读提醒：每章优先考虑一个“意外钩”。它可以是意外得物、材料、人、证据、反噬或温情旧物，最好满足三点：先让读者惊喜，再让读者意识到它有代价，最后在本章或后续3-10章变成规则、资产、债务或敌方反制。不是每章都要机械掉宝；如果本章靠情绪爆发、战斗压迫或关系变化取胜，可以说明理由。

意外钩软豁免：若本章是大战后余波、纯情绪沉篇、关系变质、战后清点、失声后的喘息，可以主动放弃“新道具/新人/新材料”。此时更适合用“旧物的新伤痕”“一句话的沉默”“一口饭没吃完”“船上少了一声”作为章末残留。读者在压力释放期不一定需要新信息，他们可能只需要确认：人还在，船还在，感情还在。

## 6. 每章读者审计

写完每章后可以快速扫一遍：

1. 普通读者能否一句话说出本章发生了什么。
2. 本章有没有人命、饭、灯、船、旧物、敌人脸、任务中的至少一项。
3. 本章是否正义一直爽，还是变成主角原地自责。
4. 本章敌人有没有聪明反制。
5. 本章是否获得或损失了一个可见资产。
6. 本章章末钩子是否具体。
7. 本章有没有术语先行。
8. 本章有没有无意义短句。
9. 本章有没有AI腔：过度工整、过度排比、空泛总结、角色像在念规则。
10. 本章是否和前后文连续。
11. 本章意外钩是什么：表面用处、真正用处、代价、回收章分别是什么。
12. 本章有没有连续三段缺少动作、物件、声音、气味或身体感。
13. 本章有没有角色只在讲道理，没有暴露私心、疼点、口癖或职业习惯。
14. 本章最像AI腔的一句是哪句，是否已经改成更像人说人事。
15. 本章新规则是否先见血、先损物、先改变局面，再给解释。
16. 本章章末是否有物理结算：多了什么、少了什么、破了什么、敌人学会了什么。
17. 本章是否有情绪杂质、环境参与或信息差悬念，至少命中一项。
18. 本章有没有一个可被读者复述的台词、动作或物件定格。
19. 本章主角、配角、敌人的选择是否智商在线：输赢是否来自信息差、代价、时间、私心和规则边界，而不是角色突然变蠢。

以上问题用于发现风险，不是逐条打勾。若某一项因当章节奏、留白、压迫感或角色状态而故意不满足，可以在审计中说明原因，不必机械补料。


## 6.1 每章完整审计输出模板

每章写完后，不要只检查“无 Markdown、字数合格、短句正常”。这些只是发布格式检查，不等于创作审计。

每章审计可参考以下八块。正式写章和重修关键章尽量完整使用；轻微修订、短段调整时可以压缩，但优先别跳过“读者是否读得顺、人物是否像人、章末是否有追读”这三个判断。

1. 八格驾驶舱：危险、目标、敌压、行动、战利品、感官、关系、钩子。
2. 意外钩审计：获得什么、谁带来、表面用处、真正用处、代价、回收章、情绪推进。
3. 基础发布检查：字数、纯 TXT、无 Markdown 误入、短句是否有效、番茄复制是否方便。
4. 读者层检查：快读读者、听书读者、情绪读者、设定党、老书虫、反感 AI 读者是否能读进去。
5. 核心检查：开章五秒、三分钟目标、八分钟奖励、章末三钩、爽点或情绪落点、可见资产或损失。
6. 创作质量检查：活人感、去 AI 化、阅读流畅度、生活细节、非漂亮话、人物是否像活人而不是规则容器。
7. 连续性检查：是否承接上一章，是否为下一章留下需要照看的债务，人物能力是否错配。
8. 主创主动升级：本章新增了什么创意，哪里可能俗，哪里还能更爽，是否需要立刻改正文。

若审计发现影响追读、人物真实感或连续性的实质瑕疵，优先先改正文，再进入下一章。若只是可接受的小偏差，可以记录原因和后续观察点，避免为了满足表格把正文修僵。

## 7. 每10章审计

建议每10章通读一次；若发现影响连续性、追读或人物真实感的瑕疵，先修再继续。

检查：

- 剧情是否连续。
- 角色能力是否错配。
- 旧设定是否误入。
- 爽点是否轮换。
- 有没有连续内耗。
- 有没有术语过密。
- 有没有重复章名结构。
- 有没有资产结算。
- 有没有新钩子。

追加五问：

1. 这10章有没有读者能记住的名场面。
2. 这10章有没有敌方经济诱惑。
3. 这10章有没有新收集物。
4. 这10章有没有一个角色主动欲望变强。
5. 这10章有没有至少一次正义打穿敌方规则的爽点。

追加三条人味儿复盘：

1. 找出这10章最像AI腔的三句，优先当场改掉或写入禁用提醒。
2. 找出这10章最有人味儿的三个动作，后续复用其写法而不是复用原句。
3. 检查主要角色是否出现口吻趋同；若趋同，下一章优先用职业动作和不漂亮的话拉开。

## 8. 百万读者模拟规则

这是模拟读者团，不是真实读者。

每个大阶段按以下读者层过滤：

- 快读爽文读者：看是否够刺激、够清楚、够想翻页。
- 听书读者：看是否听得懂人物、目标和战果。
- 设定党：看体系是否新。
- 情绪读者：看妹妹、伙伴、普通人是否有情绪抓手。
- 商业读者：看章末是否有追读欲。
- 老书虫：看是否套路过熟。
- 反感AI读者：看是否有机械味、概念味、空泛味。

若七类读者中有三类以上排异，优先先改影响最大的入口、目标或章末钩。

## 9. 市场与查重

不得照搬任何已有作品的世界观组合、主角能力、章节桥段和标志性名场面。

本书独特组合要保持稳定：

地球被命名为舟材世界 + 未名舟 + 抢回命名权 + 妹妹燃灯童 + 船体养成 + 承约副本 + 军团组合 + 正义规则扩张。

检查相似作品时，不只查词，还要查结构：开局灾难、系统能力、伙伴收集、组织成长、反派逻辑、长篇升级路线是否同构。

## 10. Codex 主创主动审计

每章写完后，除常规审计外，Codex 优先额外看一眼：

1. 本章有没有我主动新增的创意，而不是只执行用户已有想法。
2. 本章有没有一个能让读者眼睛亮一下的陌生感。
3. 本章有没有一个可长期复用的基建资产。
4. 本章有没有一个可传播的名场面、梗、物件或台词。
5. 本章有没有一个敌人更聪明的新打法。
6. 本章有没有一个角色更活的新动作。
7. 如果我现在是严苛主编，我会要求本章再升级哪里。

若以上七问有三项以上答不出来，优先补一个能立刻改善追读的创意、基建或爽点，再进入下一章。

## 11. 开章五秒检查

每章开头前五秒，优先让读者知道至少一项：

- 谁快死了。
- 什么东西快被拆了。
- 哪个名字快被改掉。
- 哪条路快关闭。
- 哪个敌人来了。
- 哪个选择来不及了。

如果开头先解释背景，优先重看入口，补一个危险、异常、欲望或上章回声。

## 12. 章末三钩规则

章末优先命中一钩：

1. 危险钩：敌人来了、倒计时变短、某人被标价。
2. 奖励钩：新船舱亮了、新伙伴醒了、新物件到手。
3. 反转钩：刚赢的东西被敌人拿去反用，或被救者提出意外选择。

最好一章末尾同时有奖励和危险。只留抽象命题时，优先补一个物件、动作、声音或敌方反应。

## 13. 情绪恨点检查

开局和每个新阶段，优先让读者看到敌人具体怎么坏。

坏不能只停留在概念，如“剥夺命名权”。要写成可见羞辱、可见价格、可见拆解、可见诱惑。

每50章尽量有一次敌方公开羞辱或公开标价，让读者重新恨敌人。

## 14. 赢大局输小局检查

每卷至少一次：主角方赢下大局，但输掉一个扎心小局。

观察：

1. 大局要有爽感，别只剩憋屈。
2. 小局要有疼感，别无关痛痒。
3. 小局要成为后续复仇、救援或角色成长燃料。

## 15. 反派升维检查

每50章末，敌人最好学会、偷走、污染或反用未名舟的一项规则。

下一循环尽量兑现敌方升级，别只写在审计里。

## 16. 普通人高光检查

每50章尽量至少一次普通人高光。观察标准：

1. 不是主角安排。
2. 不是能力者替代。
3. 行动改变战局。
4. 读者能记住这个人的动作或一句话。

## 17. 情绪缓冲带检查

每50章尽量有1-2章低压日常或半日常章节。若阶段太紧，也可以用半页温情、一个笑点或一件旧物的新伤痕承担缓冲。

观察标准：

1. 有烟火气，不是纯设定。
2. 有角色私密锚点推进。
3. 有一个可在后续大战回收的温情物件。
4. 结尾仍有钩子。

低压章不等于水章。它是下一次爆发的情绪蓄电。

## 18. 命名感官检查

每章涉及命名、抢笔、写骨、定名、拆名时，优先呈现一种具体感官，但按场面规模调节强度。小规模抢回一笔、打断一枚钉，不必每次都写成卷末异象。

可选：

- 视觉：淡金铭文膜、锈迹、灯芯纹、黑封条。
- 听觉：钢笔刮纸、印章落下、旧名低语、船骨低鸣。
- 触觉：烧红铁丝般刺痛、断钉烫掌、木纹爬肤。
- 嗅觉：焦味、火油味、旧纸味、药箱木屑味。

避免只写“他看见裂缝”“他抢回一笔”。

感官不要为了漂亮而堆满。若这段本来靠对话、动作或旧物成立，轻轻一烫、一声纸响、一点焦味就够。

## 19. 角色私密锚点检查

每个主要角色每30-50章尽量回收一次私密锚点。

不是每次都要大情节，可以是一句话、一个动作、一件小物。

若角色连续50章只履行功能，没有私密锚点，本角色读感会变成工具人，优先补一个小动作、小物件或不漂亮的话。

## 20. 沈砚深度检查

沈砚不能只做难度工具人。

每次他出场，优先满足一项：

1. 证明合法命名短期有效。
2. 展示他曾是守灯人的旧伤。
3. 用温和态度做残酷决定。
4. 让卿宸意识到：打败沈砚不是证明他蠢，而是证明“错误的名字也比没有名字强”这套哲学错了。

## 21. 正文写作驾驶舱

审计规则很多，但真正写正文时，先只看这一页。

### 开写前八格

1. 危险：本章谁快失去什么。
2. 目标：本章要抢、救、守、骗、换、逃的东西是什么。
3. 敌压：敌人主动做了什么，不是只等主角行动。
4. 行动：主角方用什么具体动作推进，不靠解释。
5. 战利品：本章得到或失去什么可见物。
6. 感官：命名、抢笔或船体变化用哪一种感官呈现。
7. 关系：本章至少一组人物关系有什么微小变化。
8. 钩子：章末是危险钩、奖励钩还是反转钩。

如果八格空得厉害，先补纲，别急着写正文。

## 22. 叙事视角规则

正文默认使用贴近卿宸的第三人称有限视角，让读者跟着他看见、听见、误判和反应。

允许短暂切到反派或普通人，但优先满足：

1. 不超过本章篇幅的三分之一。
2. 切视角要带来危险、误判、情绪或反转。
3. 不用上帝视角解释世界观。

每个场景只选一个感官中心。不要同一段里让所有角色都像作者一样清醒。

## 23. 术语配额

001-030每章最多重点使用两个新术语。

术语出现顺序优先是：物件先行，动作跟上，名称最后落地。

例如先写木牌挂上船头，再让人说这是求救木牌；先写笔画像烧红铁丝被抽出，再叫抢笔。

若一章需要解释三个以上术语，说明这一章基建过载，可以拆章或改成动作呈现。
