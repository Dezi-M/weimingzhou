# 《未名舟》正文写作与审计手册

本文件记录当前正文写作与审计口径，只服务正在推进的新大纲。

## 0. 使用口径

本表不是逐条扣分表，而是创作诊断工具。

执行时分三层：

1. 红线：读者看不懂、人物不像人、章节没有推进、正文格式不适合发布。这类问题优先修。
2. 诊断：人味、意外钩、感官、私密锚点、爽点轮换。发现缺口后先判断是否真的影响追读。
3. 建议：具体频率、数量、模板、复盘项。可按章节类型灵活取舍。

如果某条规则让正文变僵，优先相信场面和人物，再回头调整规则。

去AI味、听感、读感都不是“辞藻更优美”。危险、饥饿、争吵、疼痛和行动段可以直白，甚至应该直白；旧物、余波、沉默和情绪回声才更适合多一点质感。

## 1. 正文格式

正文只写纯 TXT，不混入 Markdown。

新章节正文 TXT 内不写章节标题或章名。章节标题只保留在文件名、后台标题、连续性索引和审计文档里；正文第一行必须直接进入场景。

不得使用：

- `#` 标题符号。
- `**` 加粗符号。
- Markdown 表格。
- 作者备注、HTML 注释或审计说明。

字数参考：

- 普通推进章：常见 3000-3800 字。
- 关键转折章：常见 3800-5200 字。
- 小高潮章：常见 5200-7000 字。
- 大高潮或卷尾：常见 7000-9500 字。

以上只作为听书节奏与发布节奏参考，不作为“为了凑数硬灌解释”的理由。场面压力、情绪回收、动作清晰和章末钩优先。

字数闸门：

- 每章完稿后必须同时记录 `wc_like` 和 `no_ws`。
- 普通章节 `no_ws < 3000` 视为未闭环，不得开下一章；除非用户明确说“短章保留”，并在结尾汇报中写明例外原因。
- `3000 <= no_ws < 3200` 为偏短观察区，必须说明保留理由，或补强动作、代价、人物选择、战场余波、章末物理钩。
- 补字不能灌规则解释、主题总结或重复表态。优先补会影响后文的伤、债、敌人样本、船体变化、救回的人、普通人选择。
- 若连续创作时发现上一章未过闸门，先补上一章，再处理新章。

## 2. 开写前八格

真正写正文时，先只看这一页。

1. 危险：本章谁快失去什么。
2. 目标：本章要抢、救、守、骗、换、逃的东西是什么。
3. 敌压：敌人主动做了什么，不是只等主角行动。
4. 行动：主角方用什么具体动作推进，不靠解释。
5. 战利品：本章得到或失去什么可见物。
6. 感官：命名、抢笔或船体变化用哪一种感官呈现。
7. 关系：本章至少一组人物关系有什么微小变化。
8. 钩子：章末是危险钩、奖励钩还是反转钩。

如果八格空得厉害，先补纲，别急着写正文。

## 2A. 软规则创作线

开写前不需要把所有规则背一遍，只抓一条软线：

脑洞火花 -> 单章八格 -> 正文动作链 -> 章末物理钩 -> 审后自修 -> 必要时回写账本或复盘。

每一环只问最有用的事：

- 脑洞火花：这一章有没有一个比旧方案更刺激、更有辨识度的可能。
- 单章八格：危险、目标、敌压、行动、战利品、感官、关系、钩子是否能支撑正文。
- 正文动作链：人物是否一直在做事，而不是旁白解释。
- 章末物理钩：本章落下的是物、伤、敌招、船体变化、救回的人，还是抽象总结。
- 审后自修：发现影响追读的硬问题，先改正文，不把问题留给下一章。
- 回写进化：如果这是可复用问题，写进账本、伏笔索引或创作复盘。

软规则的意思不是“可做可不做”，而是它服务场面。场面热时，规则退后；场面散时，规则把路找回来。

## 3. 规则战降档，生存战斗升档

这是当前最重要的书型校准。

命名权脑洞要保留锋利度，但章节快感不能主要变成“纸面机关出题 -> 旧物拆题 -> 纸面机关记录新题”。

每章开写和审稿时，优先问：

- 这章他们缺什么：水、粮、药、床、热、船板、航路、时间，还是一个快被押运走的人？
- 这章他们抢什么：人、药柜、水柜、旧门、钟声、街口、残舟钩、敌方武器、地球组件，还是一条能活下去的路？
- 慢一步会失去什么：伤者断气、卿禾灯芯变薄、船头偏向白灯、旧物被归档、残舟被拖走、临时岸沉下去？
- 抢回来以后立刻改变了什么：能煮饭、能压伤、能挡风、能守舷、能开路、能拉索、能并肩、能让船多活一口气？

一句话判断：

> 读者这一章记住的最好不是“我懂了一个规则”，而是“他们差点渴死，抢回三滴水”。

## 3A. 039 后新引擎：人间开阵写法

“人间，开阵”是阶段爆点，不是每章口号。

正文原则：

1. 不先解释终相、复相、万象开阵。先写敌人压境、旧物受损、人做选择、战场改变。
2. 人是主导，旧物是证据。必须先有人做反用途动作，旧物才应阵。
3. 敌人要聪明，会拆人和旧物之间的关系，而不是站着等旧物反杀。
4. 开阵要有失日债。锅、床、校牌、车站、门、碗、秤被迫上战场后，必须留下回不去日常的伤。
5. 小战不喊完整口号，中战可半句，大结算才完整喊。

每次开阵前，先确认四件事：

- 谁被压成了唯一用途。
- 谁先做了一个不服从用途的动作。
- 哪件旧物、旧业、旧城或旧伤应了这个人。
- 赢了什么，欠了什么。

战斗结构建议：

1. 压境：敌人拿出聪明方案，让读者觉得难打。
2. 小胜：七奇锋或某个普通人撕开一个口子。
3. 反噬：敌人预判，小胜变陷阱。
4. 人心动作：一个人拿出最舍不得的东西或做最疼的选择。
5. 开阵破局：只赢关键一段，不全赢，留下更大敌人和失日债。

判断一句：

> 爽点必须带泪，泪点必须反推爽。

如果只是旧物显灵、卿宸解释、敌人被无脑碾压，本章必须重搭。

## 3B. 上帝视角防滑坡

039 后每章审稿时，额外看五个滑坡点：

1. 是否把“人间开阵”写成新术语，而不是写成抢人、挡脚、扣影、救伤、断门、开路。
2. 是否把阶段奖励拖太远。本章哪怕输了，也要给读者一个新样本、新战法、新伤口、新关系或新敌方误判。
3. 卿宸是否有主角动作。他可以克制，但不能旁观；至少要挡、看、错开、递回、压住或亲自承受一次。
4. 敌人是否在学习。上一章吃过的亏，下一次要被它利用、污染、伪装或反设陷阱。
5. 奇锋和军团是否混用。奇锋是小队破点，军团是大面战线；不要把小队写成低级军团，也不要让军团做只有三个人潜入才能做的活。

如果五项里有两项空，本章不是润色问题，而是发动机问题，优先重搭场景。

## 4. 人味、流畅与去AI化

角色不能像规则容器。

主要出场人物尽量至少有一种活人表现：

- 身体反应：饿、冷、疼、困、手抖、嘴硬、忍笑、喘不上气。
- 小动作：藏东西、摸旧物、挡一下、退半步、嘴上骂手上帮。
- 不漂亮的话：急了说错话、怕死、嘴碎、护短、算小账。
- 私人判断：不是因为大道理，而是因为他在乎某个人、某口饭、某件旧物。

正文要让手机读者一滑就懂，听书读者一听就跟上：

- 大约每300-500字落一次具体动作、物件、声音、气味或身体感。
- 避免连续三段全是解释、判断、总结。
- 设定尽量从动作里出来：先看见人被改、饭被锁、船被拖，再说规则。
- 对话要推进局面，不能只是互相解释世界观。
- 每章尽量有一个“读者能复述的朴素句子”。

## 4A. 顺读、少字与情绪保温

新章完稿后，不只跑红旗词，还要做三遍顺读。

第一遍：句子顺读。

- 逐句低声读，看能不能自然说出口。
- 查主语丢失、谓语断裂、搭配别扭、指代不清、少字漏字、前后半句咬不上。
- 一句话如果塞了三件事，优先拆成动作、反应、结果。
- 读到要回头才能懂的句子，先改句子，不先解释设定。

第二遍：段落接气。

- 每段第一句要接住上一段留下的动作、物件、声音、伤口或未说完的话。
- 避免两个好段落硬贴在一起，中间只靠一句漂亮判断。
- 如果段落像拼接，优先补一个身体反应、物件变化、敌人动作或人话。
- 场面跳转要让读者知道人在哪里、手里有什么、眼前危险是什么。

第三遍：情绪保温。

- 每段都要有一个情绪方向：急、怕、疼、怒、松、燃、酸，不能只有信息。
- 爽点前先让读者知道谁会疼、谁会失去什么；泪点不喊感动，写舍不得和代价。
- 番茄章节要让读者每滑一屏都有下一口气：新危险、新动作、新选择、新物件或一句扎人的话。
- 高压段可以短句密集；喘息段要有中句接住，让听书不一直断拍。

AI味和拼接感的常见症状：

- 单段都漂亮，但段与段之间没有呼吸。
- 先总结“这意味着什么”，再补动作。
- 人物刚害怕就立刻燃起来，中间缺一口疼、一句错话或一次选择。
- 所有人都像分析员，台词只负责解释局面。
- 反复使用“不是X，而是Y”“他终于明白”“这一次真正”等总结腔。
- 章末用主题升华收束，而不是落在物、伤、声音、敌招或选择后果上。

直接改法：

1. 把“他意识到/这说明/这意味着”改成一个动作或一句人话。
2. 把连续解释拆成“敌人动一下 -> 旧物变一下 -> 人疼一下 -> 局面变一下”。
3. 把孤立金句接回上一段的物件、伤口或声音。
4. 把太工整的排比打散，让角色按脾气、职业和当下疼痛说话。
5. 把少字漏字当发布事故处理；读不顺就改，不用等用户指出。

## 4B. 外部取样后的去AI软信号

这一节不是硬规则，不做逐条打勾。它只在正文读起来“太正确、太整齐、太像被修平”时提醒主创回到场面。

最该补的不是口语词，而是人的不稳定：

- 人物可以说错、算错、赌错、嘴硬、护短、后悔半息，再把选择做下去。
- 配角不要只负责理解主角。让他先护自己的碗、药、亲人、面子、旧职业，再被局面推着站出来。
- 敌人不要只负责证明主角聪明。让敌人有自己的验货压力、买家压力、庭内升迁和上次吃亏后的反制。
- 主角不要每次都立刻说清意义。卿宸可以先挡、先错开、先压住，等场面给出结果。

最该避开的不是某个词，而是过度平整：

- 情节太单轨：每一步都像为了走向正确答案，缺少误会、失败、私心、旁枝和反咬。
- 主题太明白：每个场面都替读者总结“我们反抗的是定义”，读者反而失去自己悟到的快感。
- 情绪太干净：害怕就纯害怕，热血就纯热血，牺牲就纯高尚。更像人的写法是怕里有贪、燃里有疼、舍里有抱怨。
- 修稿太光滑：把粗话、错话、小算盘和短促反应全修掉，只留下漂亮句子。

让正文更像人写，不靠故意写错，也不靠加网络口头禅。优先加这些东西：

- 场内限制：时间不够、手受伤、船在偏、药只剩半瓶、敌人已经学过一次。
- 信息差：有人不知道全貌，有人误会卿宸，有人只看见自己那口锅或那张病床。
- 非功能反应：摸伤口、心疼碗、抢半句骂、藏一粒米、下意识看亲人。
- 不完整答案：本章赢一段，留下一枚钉、一笔债、一处误判或一个没救回来的人。
- 角色专属词路：老周从饭进，苏晚从伤进，铁牙从线进，陆小满从字进，梁春枝从秤进。

外部经验里需要丢掉的部分：

- 不为了“像真人”故意制造语病、废话或错别字。
- 不把去AI变成同义词替换、句式随机化或机械加口语。
- 不把每章修成同一套“短句 + 金句 + 反转”的模板。
- 不因害怕 AI 味就放弃清楚；番茄读者首先要听懂谁在抢什么、慢一步会失去什么。

## 4C. 全网AI写作教训的反踩坑复核

外部经验只当提醒，不当裁判。每次修稿前先问一句：这次修改是在让故事更有事件、选择和代价，还是只让文字看起来更像“被认真修过”。

可吸收的教训：

- 风格压过故事：如果一次修改只改变措辞，没有改变动作、选择、伤口、战果或敌方反制，先怀疑它只是表面润色。
- 创意只停在壳上：新名词、新口号、新组织不算真正新鲜；真正有用的是一个陌生的困境、反常的用途、意料外的人心选择。
- 世界状态会漂：长篇最怕今天赢了，明天像没赢过；每章至少留下一个会影响后文的物、伤、债、人、敌方样本或航道变化。
- 精致但空：段落如果删掉后，人物目标、情绪变化、战场位置和后续因果都不受影响，它大概率只是漂亮填充。
- 检测器不可迷信：不要为了通过或躲避 AI 检测器改文。检测只能提示风险，真正的标准是读者能否跟住人、事、物、痛和钩子。
- 模式会自我复制：如果最近几章都是“敌压落下 -> 卿宸看破 -> 旧物回应 -> 文书记录”，下一章必须换成追击、抢救、埋伏、登船、分饭、修船、换药或敌人亲自落刀。

复核时只抓最伤读感的两三处。若场面本来已经活，不要为了证明“我审过”继续打磨到失血。

## 5. AI腔软雷达

以下情况出现两项以上，优先回看；如果已经影响顺读和人物真实感，再重写：

- 连续排比，句式过齐。
- 用抽象词替代画面，如“秩序崩塌”“情绪爆发”“规则升级”，但没有人、物、动作。
- 角色说话像总结报告。
- 段尾总是拔高主题。
- 主角频繁自我审判，却没有立刻变成行动。
- 旁白替读者解释“这意味着什么”，而不是让场面自己说明。
- 每个人说话都同一种口吻。
- 章末强行升华，没有物件、动作、声音或选择后果。

修稿顺序：

1. 把抽象判断换成具体动作。
2. 把旁白解释换成角色选择。
3. 把漂亮台词换成带脾气、带职业、带饥饿或疼痛的话。
4. 把总结段拆进饭、灯、船、伤口、旧物、敌人动作里。
5. 把“因为X所以Y”改成“X触到实体 -> 实体变化 -> 人被迫反应 -> 局面改变”。

## 6. 角色口吻

角色台词要长在职业、旧物和脾气里。

- 老周尽量从锅、饭、碗说起。
- 苏晚从伤口、位置、药和记录说起。
- 铁牙从守线、打结、旧役说起。
- 陆小满从木牌、铅笔、记账说起。
- 云织从针脚、门、线说起。
- 孙福来从钥匙、门缝、后路和旧城说起。
- 梁春枝从秤、账、斤两和不公平说起。

配角不要被写成提问机器。可以问错、怕错、站错一息，但要有自己的职业判断和利益算盘。

## 7. 情绪呼吸与笑点

《未名舟》不能只有高频爽点，也要有让读者想住在船上的日常温度。

情绪缓冲不等于水章。它是下一次爆发的情绪蓄电。

可用呼吸方式：

- 半页分水、补锅、换药、缝衣、修门。
- 角色嘴硬、错话、小算盘、职业习惯。
- 旧物尴尬：锅盖漏、钥匙卡、断秤偏、碗不够大。
- 饭香、药味、旧校牌、错字本、乡愁面。

幽默口径：

- 不要写段子。
- 不要冲淡死亡、背叛、名丢失或新牺牲。
- 笑点来自人还活着：老周嘴硬，梁春枝损人，孙福来钥匙先露怯，邱成心疼碗，孩子认错字，旧工具在最糟糕的时候掉链子。

## 8. 命名感官

每章涉及命名、抢笔、写骨、定名、拆名时，优先呈现一种具体感官，但按场面规模调节强度。

可选：

- 视觉：淡金铭文膜、锈迹、灯芯纹、黑封条。
- 听觉：钢笔刮纸、印章落下、旧名低语、船骨低鸣。
- 触觉：烧红铁丝般刺痛、断钉烫掌、木纹爬肤。
- 嗅觉：焦味、火油味、旧纸味、药箱木屑味。

避免只写“他看见裂缝”“他抢回一笔”。

感官不要为了漂亮而堆满。若这段本来靠对话、动作或旧物成立，轻轻一烫、一声纸响、一点焦味就够。

## 9. 每章爽感软检

每章优先命中两项；若是低压日常、情绪余波或必要过渡章，可以用强情绪、强关系变化或强章末钩替代。

- 抢名。
- 救人。
- 打脸。
- 反杀。
- 收伙伴。
- 开船舱。
- 得旧物。
- 接承约。
- 分账。
- 军团组合。
- 敌人阶段胜利。
- 普通人主动选择。
- 船体升级。
- 地图奇观。
- 人间开阵。
- 敌人聪明反制后被艰难打穿。

若只命中“解释规则”，本章容易空，优先改成动作、选择或可见战果。

每章尽量明确一个“本章战利品”或“本章损失物”。它可以很小，比如一笔真名、一枚断钉、一张旧票、一碗粥、一条航迹、一个人转身留下。

## 10. 章末三钩

章末优先命中一钩：

1. 危险钩：敌人来了、倒计时变短、某人被标价。
2. 奖励钩：新船舱亮了、新伙伴醒了、新物件到手。
3. 反转钩：刚赢的东西被敌人拿去反用，或被救者提出意外选择。

最好一章末尾同时有奖励和危险。只留抽象命题时，优先补一个物件、动作、声音或敌方反应。

## 11. 每章读者审计

写完每章后快速扫：

1. 普通读者能否一句话说出本章发生了什么。
2. 本章有没有人命、饭、灯、船、旧物、敌人脸、任务中的至少一项。
3. 本章是否正义一直爽，还是变成主角原地自责。
4. 本章敌人有没有聪明反制。
5. 本章是否获得或损失了一个可见资产。
6. 本章章末钩子是否具体。
7. 本章有没有术语先行。
8. 本章有没有 AI 腔。
9. 本章是否和前后文连续。
10. 本章意外钩是什么：表面用处、真正用处、代价、回收章分别是什么。
11. 本章新规则是否先见血、先损物、先改变局面，再给解释。
12. 本章章末是否有物理结算：多了什么、少了什么、破了什么、敌人学会了什么。
13. 本章主角、配角、敌人的选择是否智商在线。
14. 若涉及开阵，是否是人先动、物再应，而不是旧物自动开挂。
15. 本章胜利是否留下失日债，而不是白拿爽点。
16. “人间，开阵”是否只用在足够大的阶段爆点上。
17. 本章句子是否能低声顺读，是否有吃字、少字、漏字和指代不清。
18. 本章段落是否接气，是否有好段落硬拼、金句贴片和突然跳情绪。
19. 本章情绪是否保温，是否让读者持续知道该急谁、疼谁、恨谁、爽在哪里。
20. 本章去 AI 化是否落在动作、人话、物件和代价上，而不是只把句子改漂亮。
21. 本章是否保留了误会、私心、毛边和信息差，而不是把人物都修成正确发言人。
22. 本章是否让敌人也在学习、承压和反制，而不是只给主角当证明题。

以上问题用于发现风险，不是逐条打勾。

但字数闸门不是软风险。若 `no_ws < 3000` 且没有用户明确批准短章，本章不算完稿，不能进入下一章。

## 12. 完整审计输出模板

正式写章和重修关键章可参考八块：

1. 八格驾驶舱：危险、目标、敌压、行动、战利品、感官、关系、钩子。
2. 意外钩审计：获得什么、谁带来、表面用处、真正用处、代价、回收章、情绪推进。
3. 基础发布检查：字数、纯 TXT、无 Markdown 误入、短句是否有效、番茄复制是否方便。
4. 读者层检查：快读读者、听书读者、情绪读者、设定党、老书虫、反感 AI 读者是否能读进去。
5. 核心检查：开章五秒、三分钟目标、八分钟奖励、章末三钩、爽点或情绪落点、可见资产或损失。
6. 创作质量检查：活人感、句子顺读、吃字少字、段落接气、情绪保温、去 AI 化、生活细节、非漂亮话、人物是否像活人。
7. 连续性检查：是否承接上一章，是否为下一章留下债务，人物能力是否错配。
8. 主创主动升级：本章新增了什么创意，哪里可能俗，哪里还能更爽，是否需要立刻改正文。

若审计发现影响追读、人物真实感或连续性的实质瑕疵，优先先改正文，再进入下一章。

## 12A. 审后自修与自主决策权

审计不是写评语，是给正文动刀。

默认权限：

1. 句子级问题：直接改。包括 AI 腔、解释过长、口吻不稳、动作不清。
2. 场景级问题：可以重搭。包括开头没压力、敌人不主动、反杀太容易、章末太虚。
3. 章节级问题：可以重写发动机。包括本章主菜变成规则说明、卿宸旁观、普通人没选择、胜利无代价。
4. 阶段级问题：先暂停继续写，回到 `03` 重排五章或十章路线。
5. 地基级问题：回到 `00/04/05` 改主控、能力或审计规则，不把坏规则继续执行。

什么时候不自行改：

- 用户明确说“只审不改”。
- 两条路线都能成立，且会明显改变主线口味。
- 需要决定是否推翻已发正文。
- 涉及用户明确保留的角色、设定、章节事实。

审后最小闭环：

1. 发现问题。
2. 判断层级。
3. 改正文或改章纲。
4. 再扫一次入口、动作链、章末钩。
5. 只把会影响未来续写的事实写回账本。
6. 若问题会反复，写进创作复盘，让下一次自动避开。

一句话：

> 能修句子就修句子，句子救不了就重搭场景，场景救不了就重写本章，章节救不了就改路线。

## 12B. Review 本身的去AI复核

审稿也会 AI 化：看起来全面，实际只把读者问题翻译成一堆漂亮条目。

每次写 review 前，先做一次反审稿：

1. 这份 review 有没有指出最影响追读的 1-3 个问题，而不是平均覆盖所有规则。
2. 每个问题能否落到正文动作：补哪口气、删哪段解释、换哪个物件、让谁先做什么。
3. 有没有写清楚“这一章要保留什么”，避免越改越平。
4. 有没有把外部经验照搬成硬规定。如果是，改成《未名舟》的场内动作。
5. 有没有为了显得专业而变长。用户要复查时，结尾列简表；正文判断要短、准、能动刀。

Review 的目标不是证明检查过，而是让下一版正文更好读、更疼、更想追。

## 12C. 正文事实同步闸门

正文 TXT 是唯一事实源。账本、入口、连续性索引、伏笔索引、素材钩表和 review 都是衍生文档，不能反过来替正文定事实。

每次新章完成、补章、重修章末钩或改动战果/代价后，必须从正文反推一次事实链：

1. 本章核心动作：谁为了什么冒险，慢一步失去什么。
2. 明面战果：赢了什么、救了什么、挡住什么。
3. 可见代价：伤、裂、断、少、沉、记忆损耗、敌方采样。
4. 敌方学习：敌人带走了哪个样本，下一章能怎么反咬。
5. 章末钩：下一章入口落在哪个物、伤、声、门、债或敌招上。

反推后同步检查这些位置：

- `docs/AI扫描入口.md`：当前硬状态、最新章事实、下一章方向。
- `docs/14_当前故事状态账本.md`：最新事实、阶段拍点、活债。
- `docs/AI章节连续性索引.md`：章节功能表、当前活债。
- `docs/伏笔索引.md`：仍会反咬的物件、伤口、敌方样本。
- `docs/06_039后新素材库_人间开阵.md`：已写章节的素材钩行；未写章节可以是计划，已写章节必须贴正文。
- `reviews/`：若 review 内有字数、钩子、代价或下一章建议，不能保留旧结论。

禁止用“搜不到旧关键词”代替事实同步。必须人工对比：小奖/损失、章末钩、代价、敌方样本、下一章入口是否和正文一致。

一句话：

> 写章时文档给方向；章写完后，文档听正文的。

## 12D. Token 节流与质量保护

省 token 的原则不是少创作、少审计，而是少做不会改善正文的重复动作。

默认分三档：

1. 轻量高质档：普通续写使用。只读入口、账本、最新章尾、当前关键词锚点；写完后做字数、纯 TXT、正文内无章名标题、红旗词、`git diff --check` 和必要文档回写。
2. 深审档：关键章、小高潮、用户要求“审一下/AI味/发布前”、轻量档发现实质问题时使用。补读 `05` 的 4A-4C、11、12A-12C，必要时读 `AI章节连续性索引` 和 `伏笔索引` 相关段。
3. checkpoint 档：五章、十章、发布包或路线调整时使用。允许扩大到多章字数、正文内无章名标题、连续性、重复压力和素材钩全量核对。

普通单章不要做这些高耗低益动作：

- 不用 `nl -ba` 打印整章；只抽查开头、中段代价处、章末钩。除非准备逐段重修。
- 不把 `docs/`、`reviews/`、`chapters_txt/` 全量扫描当默认动作；只扫本轮改动文件和当前章相关关键词。
- 不反复读取同一大文档；第一次读取后，只用 `rg` 或 `sed` 定位需要的小节。
- 不在 final 里堆探索命令；只列真正形成检查结论的命令。

绝不能为了省 token 跳过这些：

- `no_ws < 3000` 的硬闸门。
- 章节顺读、少字漏字、段落接气、情绪保温。
- 敌人是否聪明反制。
- 主角是否有不可替代动作。
- 普通人是否做选择并付代价。
- 章末是否落在物、伤、敌招、救回的人、船体变化或下一章物理钩。
- 事实回写是否贴正文。

触发升级深审的信号：

- 开头不是接上一章物理钩，而是在解释。
- 中段连续三段都像规则说明。
- 角色台词只负责分析局面。
- 敌人为了让主角赢而放慢或降智。
- 章末只有概念，没有可见状态变化。
- 命令检查通过，但读起来仍吃字、生硬、像拼接。

## 13. 每五章/十章/五十章

每五章看重复：

- 最近五章有没有同一种敌压、同一种解法、同一种文书结尾。
- 若重复，下一章主动换菜：资源、追击、残舟、日常喘息、主角压场、普通人高光。

每十章通读：

- 剧情是否连续。
- 角色能力是否错配。
- 爽点是否轮换。
- 有没有资产结算。
- 有没有新钩子。
- 有没有读者能记住的名场面。
- 有没有敌方经济诱惑。
- 有没有一个角色主动欲望变强。

每五十章卷审：

- 商业循环是否完整。
- 反派是否升维。
- 七类资产是否结算。
- 普通人是否改变战局。
- 赢大局输小局是否成立。
- 下一循环主菜是否清楚。

## 14. 百万读者模拟

这是模拟读者团，不是真实读者。

每个大阶段按以下读者层过滤：

- 快读爽文读者：看是否够刺激、够清楚、够想翻页。
- 听书读者：看是否听得懂人物、目标和战果。
- 设定党：看体系是否新。
- 情绪读者：看妹妹、伙伴、普通人是否有情绪抓手。
- 商业读者：看章末是否有追读欲。
- 老书虫：看是否套路过熟。
- 反感 AI 读者：看是否有机械味、概念味、空泛味。

若七类读者中有三类以上排异，优先先改影响最大的入口、目标或章末钩。

## 15. 主创主动审计

每章写完后，Codex 额外看一眼：

1. 本章有没有主动新增的创意，而不是只执行已有想法。
2. 本章有没有一个能让读者眼睛亮一下的陌生感。
3. 本章有没有一个可长期复用的基建资产。
4. 本章有没有一个可传播的名场面、物件或台词。
5. 本章有没有一个敌人更聪明的新打法。
6. 本章有没有一个角色更活的新动作。
7. 如果我是严苛主编，我会要求本章再升级哪里。

若以上七问有三项以上答不出来，优先补一个能立刻改善追读的创意、基建或爽点，再进入下一章。
