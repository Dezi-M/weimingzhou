# 《未名舟》文档使用索引

本文件只解决一件事：避免创作时把整套文档都背上身。

默认口径：

- `skill` 负责流程、路由、软审计、自进化。
- `docs` 负责世界观、大纲、阶段状态、项目记忆。
- 普通“继续”不要全量加载文档。先读最新章节尾巴、当前故事状态账本、当前阶段补纲和创作复盘；其他文档只在需要时用 `rg` 拉相关段落。
- 归档不是废弃，只是不参与日常创作指挥。

## 日常加载

| 文档 | 用途 | 什么时候读 |
|---|---|---|
| `README_文档使用索引.md` | 文档路由入口 | 不确定该读哪份文档时，先看这份。 |
| `14_当前故事状态账本.md` | 最新正文事实、角色状态、敌方样本、短期旧账 | 普通续写、断点恢复、判断下一章该接哪里时读。写完新章若状态变化，顺手更新。 |
| `12_031-050阶段补纲_拒名船悬赏榜.md` | 当前 031-050 小循环航向图 | 普通续写、判断下一章方向、照看当前旧账时读。进入新阶段后，用新的阶段补纲替代它。 |
| `创作复盘_常见问题与改进.md` | 本项目实时手感记忆 | 每次续写、修读感、修听感、修入口和回声时优先读。它比旧流程文档更贴近当前问题。 |

日常“继续”建议只带这四类上下文：

1. 最新 1-2 章结尾。
2. 当前故事状态账本。
3. 当前阶段补纲。
4. 创作复盘中最近、最相关的 1-3 条。

## 节点顺序

写到五章、十章、五十章节点时，先处理节点，再继续新章。顺序为：

1. 最新章若还没软审，先补单章软审。
2. 五章节点：跑发布包、标题/简介/听书节奏、Git 检查点。
3. 十章节点：跑连续性、资产代价、旧账折痕复盘。
4. 五十章节点：跑卷级商业循环、反派升维、资产结算审计。

如果新章已经越过节点，例如 041 已写但 040 还没发布包/十章复盘，先处理 036-040；041 留在下一批，不混进旧批次。

## 轻口径

当文档之间冲突时，按这个顺序取舍：

1. 正文已经成立的事实。
2. 当前故事状态账本。
3. 当前阶段补纲。
4. 创作复盘里的最新手感。
5. 世界规则与角色能力文档。
6. 总纲和长线设定。
7. 历史归档文档。

一句话：让文档帮正文松开，不要让文档把正文拧紧。
