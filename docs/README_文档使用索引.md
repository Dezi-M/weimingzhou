# 《未名舟》文档使用索引

本索引是当前文档目录入口。AI / Codex / 自动审稿开工时，先读 `AI扫描入口.md`，再回到本索引判断加载范围。新大纲是唯一方向盘，文档只保留能继续写正文的活内容。

## 当前最高口径

一句话：

> 地球被拆成货，卿宸开船去抢回家。

更深一层：

> 命名庭偷的不是旧物本身，而是地球和万界那些还没结束、还能改口的明天。全书底层战争是终名 vs 复名。

当前主创口径：

> 高位面把万物压成唯一用途。地球把万物打回万种活法。
> 人先不跪，物才应阵。人间，开阵。

当前已经进入全书新版重构：旧版 001-038 已从活工作区清理，旧章和旧审计不再作为事实源、素材库或连续性地基。新版从 001 重新生长；039-050 已移入 `_样本_039-050_人间开阵/`，仅作为新引擎样本，后续按新版 038 章末钩重写或重接。

新版 001-038 重构默认只读最小开写包：

1. `AI扫描入口.md`
2. `14_当前故事状态账本.md`
3. `00_新版创作主控_未名舟.md`
4. `03_十卷两千章升级表.md`
5. `04_角色能力军团承约制.md`
6. `15_001-038全量重构路线.md`
7. 写完或审稿时再看 `05_正文创作硬规则与审计表.md`。

普通续写默认按“轻量高质档”执行：不全文打印最新章，不默认全量扫 docs，不重复读取大文档；但字数闸门、顺读、纯 TXT、正文内无章名标题、红旗词、`git diff --check`、事实回写不能省。关键章、发布前、五章节点或轻量检查发现问题时，再升深审 / checkpoint 档。

需要时再补读：

- 卡连续性：读 `AI章节连续性索引.md` 和 `伏笔索引.md`。
- 卡大方向：读 `00_新版创作主控_未名舟.md`，必要时补 `03/04`。
- 修读感、AI味或复盘反复问题：只看 `创作复盘_常见问题与改进.md` 最新相关条目，不作为开章必读。

## 冲突取舍

当文档之间冲突时：

1. 新版重写后的正文事实优先。
2. `AI扫描入口.md` 和 `14_当前故事状态账本.md` 优先于其他说明。
3. `00/01/03/04/05` 五份 active 主文档负责总纲、规则、路线、体系和审计；039 后素材另以 `06` 为准。
4. 039-050 当前只作样本，不得压过新版 001-038。
5. `伏笔索引.md` 只管活伏笔，不替代阶段纲。

一句话：日常创作只看活文档，缺新敌招、新战法、新素材就在新大纲里现造；旧章不保留、不背诵、不迁就。
