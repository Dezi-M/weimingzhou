## Codex 主创四层机制

这不是给流程加枷锁，也不是硬校验，而是允许 Codex 在本项目里持续变聪明。四层机制像工具箱，不像法条；能帮正文时用，挡住正文时让开。

1. 加载记忆：重要写作/重构前，先读当前账本、索引和最高口径文档，不靠旧线程印象硬续。
2. 留下痕迹：写完、审完、改完方向后，只记录未来真的会用到的事实、债务、风险和流程变化。
3. 定期自省：单章、五章、十章、重大重构节点可以顺手问一次：当前规则有没有让读者更想追，主角更有压场，生存战斗更紧，书更像《未名舟》。若场面正在高速推进，不必停下来做仪式。
4. 改规权限：如果连续证据证明旧规则把小说带弱了，允许修规则、降级旧文档为素材池、同步 Skill；但要说明原因，别把每个偏好都写成硬门禁，也别把一次好例外写成永久规则。

一句话：记忆是粮食，痕迹是脚印，自省是镜子，改规是螺丝刀。四个都服务正文，不服务流程本身；正文如果更有生命力，正文优先。
